---
title: "Android In-App Products Setup"
---

<Info>
  Creating in-app products before [granting users access](doc:service-account-key-android#7-grant-access) or [linking Google Cloud Project](doc:service-account-key-android#3-link-google-cloud-project) to the service account may cause problems with the receipt validation using Google Play Developer API. Google Play Developer API can return the following error *"The current user has insufficient permissions to perform the requested operation."*

  When encountering this issue, open the In-app products or Subscription tab respectively on the Google Play Console and make any updates. For example, edit the product description and save it.
</Info>

In order to create an in-app purchase navigate to [Google Play Developer Console](https://play.google.com/apps/publish/) and select the 'All apps' tab. Then select your app from the list.

![](/images/docs/13bfe32-Screenshot_2021-05-21_at_11.49.39.png "Screenshot 2021-05-21 at 11.49.39.png")

Select **Products**. You can select either In-app products or Subscriptions.

![](/images/docs/f79b6f2--2022-07-07-15.24.55.png "Снимок экрана 2022-07-07 в 15.24.55.png")

**1. In-app products**

Let's choose the In-app products tab. Click the **Create product** button. You have to provide Product ID, Name, and Description.

![](/images/docs/db11fd1-Screenshot_2021-05-21_at_11.56.18.png "Screenshot 2021-05-21 at 11.56.18.png")

<Warning>
  ### Note

  You can’t change the product ID after the product has been created.
</Warning>

Set the price at the bottom of the page and apply changes.

![](/images/docs/d689b3d--2022-07-07-15.31.12.png "Снимок экрана 2022-07-07 в 15.31.12.png")

<Warning>
  ### Note

  Qonversion doesn't yet support multi-quantity purchases, so leave that option unselected.
</Warning>

Finally, check the tax and compliance settings and then click **Save**.

![](/images/docs/a8bfb68--2022-07-07-15.37.41.png "Снимок экрана 2022-07-07 в 15.37.41.png")

Once you have created a product, it has an **Inactive** status. Click the **Activate** button.

![](/images/docs/64f8f2d-Screenshot_4.png) ![](/images/docs/b9a298e--2022-07-07-15.40.04.png "Снимок экрана 2022-07-07 в 15.40.04.png")

**2. Subscriptions**

Let's choose the Subscriptions tab. Click the **Create subscription** button. Provide Product ID and Name.

![](/images/docs/0001608--2022-07-08-15.54.48.png)

Now the subscription is created but it is not configured yet. There are four steps to configure it. Two of them are optional.

![](/images/docs/b65b391--2022-07-07-18.38.28.png "Снимок экрана 2022-07-07 в 18.38.28.png")

The first step is optional. You can add up to four custom strings explaining what users get when they subscribe.

![](/images/docs/05aba16--2022-07-07-18.42.27.png "Снимок экрана 2022-07-07 в 18.42.27.png")

For the next two steps, you should create base plans and offers, if necessary. The base plan contains basic information about the subscription such as duration, price, renewal type, grace period, etc.

<Info>
  You can check our [blog post](https://qonversion.io/blog/google-play-billing-library-5-0/) for the details of the new subscription model of Google Play.
</Info>

To create a base plan click **Add a base plan** either from the task list or from the base plans and offers section.

![](/images/docs/8d60e04--2022-07-08-11.15.43.png "Снимок экрана 2022-07-08 в 11.15.43.png")

Enter the base plan identifier and configure its renewal type with billing and grace periods. You may also add tags that are used to distinguish base plans from the API side. This is not required with one base plan per subscription.

![](/images/docs/9ec708d--2022-07-08-11.46.31.png "Снимок экрана 2022-07-08 в 11.46.31.png")

The last step is to set the price. Navigate to "Prices and availability" section and click **Set prices**, select regions the subscription will be available in and press **Set price**.

![](/images/docs/f82f5b6--2022-07-08-11.52.52.png "Снимок экрана 2022-07-08 в 11.52.52.png")

Enter the price and click **Update**. Save changes.

![](/images/docs/9fcb4b4--2022-07-08-12.01.38.png "Снимок экрана 2022-07-08 в 12.01.38.png")

Once you have created a base plan, it has a **draft** status. Click the **Activate** button to make it available to users.

![](/images/docs/24b2144--2022-07-08-12.06.43.png "Снимок экрана 2022-07-08 в 12.06.43.png")

Your base plan is ready to use.

![](/images/docs/7372906--2022-07-08-12.10.56.png "Снимок экрана 2022-07-08 в 12.10.56.png")

You can use the created subscription. You can also add options like a trial period or discounts to your subscriptions. This is where offers come in. Offers belong to base plans.

Click **Add offer** to create an offer.

![](/images/docs/a18e44a--2022-07-08-15.18.53.png "Снимок экрана 2022-07-08 в 15.18.53.png")

Select the base plan to which the new offer will belong and click **Add offer**.

![](/images/docs/f4a8eb2--2022-07-08-15.19.31.png "Снимок экрана 2022-07-08 в 15.19.31.png")

Specify offer identifier and select eligibility criteria. There are several options available: users who never bought this or any other subscriptions; those, who upgraded from other subscriptions; developer-determined.

You may also add tags as for the base plan.

![](/images/docs/613e8c8--2022-07-08-15.45.25.png "Снимок экрана 2022-07-08 в 15.45.25.png")

The final step is phases. You can configure up to two phases which will be used before the base plan purchasing. For example, you can add a free trial for a week and a 10% discount for the next week before the user will buy the original subscription. To create a phase click the **Add phase** button in the Phases section.

![](/images/docs/96965cc--2022-07-08-15.37.02.png "Снимок экрана 2022-07-08 в 15.37.02.png")

Choose the phase type, duration, and, if you chose discount type, prices. In the example below we create a 10% discount price for a week.

![](/images/docs/e3f7063--2022-07-08-15.39.10.png "Снимок экрана 2022-07-08 в 15.39.10.png")

Press **Apply** and **Save** to create an offer. The offer has a **draft** status. Press **Activate**.

![](/images/docs/3075e08--2022-07-08-15.52.08.png "Снимок экрана 2022-07-08 в 15.52.08.png")

After you have done all the above steps your subscription is ready to use.
