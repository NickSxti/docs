---
title: "Mixpanel"
description: "Send in-app iOS and Android subscription and purchase events to Mixpanel"
---

Qonversion sends valuable in-app revenue events, including purchases, subscriptions after free trials, subscription renewals, and even refunds to your Mixpanel account. This allows you to match your user behavior with their payment data in Mixpanel, and inform your product decisions. Qonversion validates user receipts directly with App Stores, so the revenue events are tracked even if a user does not open the app.

## 1. Set up the SDKs

1. Make sure you have Mixpanel SDK installed. If you do not have Mixpanel integration yet, please use [this documentation](https://developer.mixpanel.com/docs/implement-mixpanel#mixpanel-sdks).

2. Set Qonversion SDKs following [installing the SDKs](doc:install-sdk) guides.

3. Attribute events sent from Qonversion and events received from the Mixpanel SDK to the same user by setting the same user id to Mixpanel that you set to Qonversion SDK in the [User Identifiers guide](https://documentation.qonversion.io/docs/user-identifiers#2-user-id-for-third-party-integrations).

<CodeGroup>
  ```bash bash
  Mixpanel.mainInstance().identify(distinctId: "yourSideUserID")
  ```

  ```bash bash
  [[Mixpanel sharedInstance] identify:@"yourSideUserID"];
  ```

  ```bash bash
  MixpanelAPI mixpanel =
      MixpanelAPI.getInstance(context, MIXPANEL_TOKEN);

  mixpanel.identify("yourSideUserID");
  ```

  ```bash bash
  mixpanel.identify("yourSideUserID");
  ```

  ```bash bash
  Mixpanel.identify('yourSideUserID');
  ```

  ```bash bash
  Mixpanel.Identify('yourSideUserID');
  ```
</CodeGroup>

<Warning>
  ### Do not track any purchase events on the client side

  Qonversion tracks all revenue events, so if you track revenue events with Mixpanel SDK, you may double-count the revenue in your Mixpanel account.
</Warning>

## 2. Configure the Mixpanel Integration

1. Go to your Mixpanel [Project Settings](https://mixpanel.com/login/?next=/settings/account#projects) and copy your token:

![](/images/docs/770cc26-mixpanel_token.jpg)

2. Navigate to the Integrations section in your Qonversion project, select [Mixpanel ](https://dash.qonversion.io/app/integration/mixpanel), and provide the **Token** and Save.

<Check>
  ### Done

  Now Qonversion will start sending in-app purchases and subscriptions data to your Mixpanel account.
</Check>

## Event Payload

In case you need details about data sent to Mixpanel, follow the example below:

<CodeGroup>
  ```bash bash
  {
     "event":"subscription_started",
     "properties":{
        "distinct_id":"user_id",
        "Product Name":"product_id",
        "time":1616128486,
        "token":"",
        "$insert_id":"",
        "Product Price":9.9,
        "$append":{
           "$transactions":{
              "$amount":9.9,
              "$time":"2021-03-19T04:34:46",
              "product_id":"product_id"
           }
        }
     }
  }
  ```
</CodeGroup>
